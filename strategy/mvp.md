# MVP DTLogix

## Вступ

MVP (Мінімально життєзданий продукт) — це мінімальний набір функцій, достатній для валідації основної гіпотези: DTLogix економить 1–2 години щодня та зменшує хаос у комунікаціях.

## Детальний опис MVP

### Must-have функції

**Створення заявки:**
- створення заявки від клієнта
- перегляд списку заявок
- перетворення заявки на рейс

**Трекінг рейсу:**
- створення рейсу з базовими даними (маршрут, перевізник, клієнт, вантаж)
- перегляд списку рейсів
- фільтрація та пошук рейсів
- редагування інформації про рейс

**Автоматичні статуси:**
- статуси рейсу: loading, transit, customs, delivered
- оновлення статусу вручну
- відображення поточного статусу в рейсі
- історія змін статусів

**ETA (очікуваний час прибуття):**
- автоматичний розрахунок ETA на основі маршруту та статусу
- ручне коригування ETA при необхідності
- відображення ETA в рейсі
- історія змін ETA

**Автоматичні сповіщення клієнту:**
- автоматична відправка сповіщень при зміні статусу
- сповіщення про зміну ETA
- сповіщення про доставку
- налаштування каналів сповіщень (SMS, email)

**Зберігання фото та документів:**
- завантаження документів до рейсу (ТТН, договори, фото, митні документи)
- перегляд та завантаження документів
- організація документів за типами
- прив'язка документів до конкретного рейсу

### Nice-to-have функції (можуть доїхати V1)

**Базовий AI-модуль для аналізу відхилень:**
- виявлення аномалій у рейсах (затримки, простої, відхилення від маршруту)
- базові рекомендації щодо уникнення простоїв

**Розширені сценарії документів:**
- автоматична генерація документів
- шаблони документів
- інтеграція з електронним документообігом

**Частина інтеграцій:**
- базові інтеграції з месенджерами
- інтеграція з SMS-провайдерами

## Критерії готовності MVP

### Продуктові критерії

**Функціональність:**
- всі must-have функції реалізовані та протестовані
- час створення рейсу ≤ 3 хвилини
- автоматичні сповіщення працюють стабільно (≥ 95% доставки)
- система працює без критичних помилок (uptime ≥ 99%)

**Якість:**
- інтерфейс зрозумілий для користувача без навчання
- онбординг займає 10–15 хвилин
- документація для користувачів готова

### Бізнес-критерії

**Кількість користувачів:**
- не менше 10 активних компаній на платформі
- кожна компанія має ≥ 5 активних рейсів/місяць

**Використання:**
- 200+ рейсів/місяць на платформі
- ≥ 80% користувачів створюють рейси щонайменше раз на тиждень
- adoption клієнтського лінку ≥ 50% рейсів

**Метрики цінності:**
- економія часу підтверджена користувачами (1–2 години щодня)
- кількість вхідних дзвінків від клієнтів зменшилася на ≥ 50%
- churn rate < 5%/міс

**Комерційні метрики:**
- conversion demo → paid ≥ 20%
- середній час від demo до оплати ≤ 7 днів
- базові unit economics підтверджені (CAC ≤ $30, LTV:CAC ≥ 3:1)

### Технічні критерії

**Стабільність:**
- система обробляє ≥ 1000 рейсів одночасно
- час відгуку інтерфейсу < 2 секунди
- завантаження документів працює стабільно

**Безпека:**
- дані користувачів захищені
- резервне копіювання налаштоване
- відповідність базовим вимогам безпеки

## Гіпотези MVP

- MVP достатньо для першої комерційної версії.
- Користувачам потрібно максимум 2–3 хвилини на створення рейсу.
- Автоматичні сповіщення зменшують кількість вхідних дзвінків на 50–60%.
- Економія часу (1–2 години щодня) — головний тригер покупки.
- Простота онбордингу (10–15 хв) — ключ до швидкого adoption.

## Наступні дії / наслідки

- Створити прототип статусів і ETA для валідації з користувачами.
- Перевірити сценарії реальних рейсів з 2–3 експедиторами перед запуском.
- Налаштувати аналітику на рівні подій для відстеження метрик.
- Зафіксувати критерій переходу до V1: не менше 10 активних компаній та 200+ рейсів/місяць на платформі.
- Зібрати дані для навчання AI-модуля з перших днів MVP.
