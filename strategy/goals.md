# Цілі DTLogix

## Вступ

Цілі продукту визначають, що має бути досягнуто на різних етапах розвитку. Всі цілі прив'язані до ключових метрик з Lean Canvas та мають чіткі критерії виконання.

## Цілі на етап MVP

### Продуктові цілі

**Економія часу:**
- Ціль: користувачі економлять 1–2 години щодня
- Метрика: час, який економить експедитор (вимірюється через опитування та аналітику)
- Критерій успіху: ≥ 80% користувачів підтверджують економію ≥ 1 години на день

**Зменшення хаосу:**
- Ціль: зменшити кількість вхідних дзвінків від клієнтів на 50–60%
- Метрика: кількість вхідних звернень (дзвінки + месенджери) до та після впровадження
- Критерій успіху: зменшення на ≥ 50%

**Adoption клієнтського лінку:**
- Ціль: ≥ 50% рейсів мають активний клієнтський лінк
- Метрика: adoption клієнтського лінку (кількість рейсів з активним лінком / загальна кількість рейсів)
- Критерій успіху: ≥ 50%

### Комерційні цілі

**Кількість користувачів:**
- Ціль: 10+ активних компаній на платформі
- Метрика: MAU активних експедиторів
- Критерій успіху: ≥ 10 компаній з ≥ 5 активними рейсами/місяць

**Обсяг рейсів:**
- Ціль: 200+ рейсів/місяць на платформі
- Метрика: кількість активних рейсів
- Критерій успіху: ≥ 200 рейсів/місяць

**Конверсія:**
- Ціль: conversion demo → paid ≥ 20%
- Метрика: conversion demo → paid
- Критерій успіху: ≥ 20%

**Retention:**
- Ціль: churn rate < 5%/міс
- Метрика: churn rate
- Критерій успіху: < 5%/міс

### Технічні цілі

**Точність ETA:**
- Ціль: точність ETA ≥ 80%
- Метрика: середнє відхилення ETA (реальний час прибуття vs прогнозований)
- Критерій успіху: ≥ 80% рейсів мають відхилення ETA ≤ 2 години

**Стабільність системи:**
- Ціль: uptime ≥ 99%
- Метрика: uptime системи
- Критерій успіху: ≥ 99%

## Цілі на етап Growth (V1 → V2)

### Продуктові цілі

**Масштабування:**
- Ціль: 50+ активних компаній, 1000+ рейсів/місяць
- Метрика: MAU активних експедиторів, кількість активних рейсів
- Критерій успіху: ≥ 50 компаній, ≥ 1000 рейсів/місяць

**Покращення ETA:**
- Ціль: точність ETA ≥ 85%
- Метрика: середнє відхилення ETA
- Критерій успіху: ≥ 85% рейсів мають відхилення ETA ≤ 2 години

**Adoption AI-модуля:**
- Ціль: AI-модуль використовується ≥ 60% користувачів
- Метрика: кількість користувачів, які використовують AI-рекомендації
- Критерій успіху: ≥ 60% користувачів

### Комерційні цілі

**MRR:**
- Ціль: MRR ≥ $2000 (V1), MRR ≥ $10000 (V2)
- Метрика: Monthly Recurring Revenue
- Критерій успіху: ≥ $2000 (V1), ≥ $10000 (V2)

**Unit Economics:**
- Ціль: LTV:CAC ≥ 3:1 (V1), LTV:CAC ≥ 5:1 (V2)
- Метрика: LTV (Lifetime Value), CAC (Customer Acquisition Cost)
- Критерій успіху: LTV:CAC ≥ 3:1 (V1), ≥ 5:1 (V2)

**Retention:**
- Ціль: churn rate < 3%/міс
- Метрика: churn rate
- Критерій успіху: < 3%/міс

**ARPU:**
- Ціль: ARPU $40–60/міс на компанію
- Метрика: Average Revenue Per User
- Критерій успіху: $40–60/міс

### Технічні цілі

**Масштабованість:**
- Ціль: система обробляє 5000+ рейсів одночасно
- Метрика: кількість одночасних рейсів
- Критерій успіху: ≥ 5000 рейсів

**Adoption мобільного додатку (V2):**
- Ціль: adoption мобільного додатку ≥ 40% водіїв
- Метрика: кількість водіїв, які використовують мобільний додаток
- Критерій успіху: ≥ 40% водіїв

**Adoption API (V2):**
- Ціль: API використовується ≥ 20% компаній
- Метрика: кількість компаній, які використовують API
- Критерій успіху: ≥ 20% компаній

## Гіпотези

- Час економії — ключовий фактор retention.
- ETA + сповіщення зменшують churn до < 3%/міс.
- Користувачі реагують позитивніше на AI-підказки, ніж на ручні нагадування.
- Модель "per user" дає кращий MRR, ніж "per trip", у сегменті B.
- Компанії, що мають 40+ рейсів/місяць, переходять на комбіновану модель.
