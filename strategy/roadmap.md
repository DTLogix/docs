# Дорожня карта DTLogix

## Вступ

Дорожня карта визначає етапи розвитку продукту від MVP до масштабованого рішення. Кожен етап має чіткі критерії переходу, засновані на метриках та гіпотезах з Lean Canvas.

## Етапи розвитку

### MVP (Мінімально життєзданий продукт)

**Призначення:** Валідація основної гіпотези про економію часу та зменшення хаосу.

**Ключові можливості:**
- створення заявки
- трекінг рейсу
- автоматичні статуси (loading / transit / customs / delivered)
- ETA
- фото та документи в одному місці
- автоматичні сповіщення клієнту
- зберігання фото та документів по рейсу

**Критерії переходу до V1:**
- не менше 10 активних компаній на платформі
- 200+ рейсів/місяць на платформі
- економія часу підтверджена користувачами (1–2 години щодня)
- churn rate < 5%/міс
- conversion demo → paid ≥ 20%

---

### V1 (Перша комерційна версія)

**Призначення:** Стабільна комерційна версія з базовим AI-модулем та покращеною автоматизацією.

**Ключові можливості (додатково до MVP):**
- базовий AI-модуль для аналізу відхилень
- розширені сценарії документів
- покращена точність ETA на основі історичних даних
- клієнтський лінк з розширеним функціоналом
- базові інтеграції (SMS, email)

**Критерії переходу до V2:**
- 50+ активних компаній
- 1000+ рейсів/місяць
- точність ETA ≥ 85%
- AI-модуль використовується ≥ 60% користувачів
- MRR ≥ $2000
- LTV:CAC ≥ 3:1

---

### V2 (Масштабоване рішення)

**Призначення:** Повноцінна екосистема з мобільним додатком, API та розширеною аналітикою.

**Ключові можливості (додатково до V1):**
- мобільний застосунок для водія
- API / webhooks
- GPS інтеграції
- менеджерський дашборд з аналітикою
- розширені AI-рекомендації
- інтеграції з бухгалтерією та маркетплейсами вантажів

**Критерії переходу до Growth:**
- 200+ активних компаній
- 5000+ рейсів/місяць
- adoption мобільного додатку ≥ 40% водіїв
- API використовується ≥ 20% компаній
- MRR ≥ $10000
- LTV:CAC ≥ 5:1
- churn rate < 3%/міс

---

## Пріоритизація між етапами

### Must-have для MVP

- створення заявки
- трекінг рейсу
- автоматичні статуси
- ETA
- автоматичні сповіщення клієнту
- зберігання фото та документів по рейсу

### Nice-to-have (можуть доїхати V1)

- базовий AI-модуль для аналізу відхилень
- розширені сценарії документів
- частина інтеграцій

### V2 та далі

- мобільний застосунок
- API / webhooks
- GPS інтеграції
- менеджерський дашборд
- розширені інтеграції

## Гіпотези

- MVP достатньо для першої комерційної версії.
- Користувачам потрібно максимум 2–3 хвилини на створення рейсу.
- AI-рекомендації збільшать retention.
- Мобільний додаток критично важливий для масштабування.
- API відкриває можливості для партнерств та інтеграцій.

## Наступні дії / наслідки

- Фокус на досягненні критеріїв MVP перед переходом до V1.
- Збір даних для навчання AI-модуля з перших днів MVP.
- Планування архітектури API на етапі V1 для легкого переходу до V2.
- Регулярний перегляд критеріїв переходу на основі реальних даних.
