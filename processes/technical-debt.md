# Управління технічним боргом

## Вступ

Технічний борг — це компроміси в технічних рішеннях, які приймаються для швидкої доставки функцій, але потребують майбутнього рефакторингу.

## Основні принципи

### Коли фіксувати технічний борг

- Коли приймається технічний компроміс для швидкої доставки
- Коли виявлено проблеми в архітектурі, які блокуватимуть майбутній розвиток
- Коли є legacy код, який потребує рефакторингу

### Пріоритизація технічного боргу

Технічний борг пріоритизується через:
- Вплив на швидкість розробки нових функцій
- Ризики для стабільності системи
- Вплив на продуктивність та масштабованість

**Критерій:** Технічний борг пріоритизується тільки якщо блокує розробку нових функцій (див. [Планування → Спринти](./planning.md#що-виноситься-у-спринти)).
