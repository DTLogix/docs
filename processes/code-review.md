# Процеси code review

## Вступ

Code review забезпечує якість коду та узгодженість у команді.

## Основні принципи

### Коли проводиться code review

- Перед мержем pull request
- Для критичних змін (архітектура, безпека)

### Хто проводить review

- Мінімум один ревьюер
- Для критичних змін — 2+ ревьюери

### Критерії перевірки

- Відповідність acceptance criteria
- Якість коду (читабельність, структура)
- Відповідність стандартам та best practices
- Наявність тестів (якщо застосовно)
- Безпека та продуктивність
