# Модулі DTLogix

## Вступ

Кожен модуль DTLogix вирішує конкретну задачу в процесі управління рейсами. Модулі інтегровані між собою та працюють як єдина система.

## Детальний опис

### Модуль "Рейси"

**Призначення:** Створення, редагування та відстеження рейсів від заявки до доставки.

**Ключові дії:**
- створення рейсу з базовими даними (маршрут, перевізник, клієнт, вантаж)
- оновлення статусу рейсу (loading, transit, customs, delivered)
- редагування інформації про рейс
- перегляд історії змін
- фільтрація та пошук рейсів

**Цінність для користувача:** Всі рейси в одному місці, швидкий доступ до інформації, можливість швидко передати рейс іншому менеджеру.

---

### Модуль "Заявки"

**Призначення:** Обробка заявок на перевезення перед створенням рейсу.

**Ключові дії:**
- створення заявки від клієнта
- перегляд списку заявок
- перетворення заявки на рейс
- відміна або архівування заявок

**Цінність для користувача:** Структурований процес обробки запитів, відстеження статусу заявок, зменшення втрачених замовлень.

---

### Модуль "ETA"

**Призначення:** Автоматичний розрахунок та оновлення очікуваного часу прибуття вантажу.

**Ключові дії:**
- автоматичний розрахунок ETA на основі маршруту та статусу
- ручне коригування ETA при необхідності
- відображення ETA в рейсі та клієнтському лінку
- історія змін ETA

**Цінність для користувача:** Прозорість для клієнта, зменшення кількості запитів "коли буде доставка", підвищення довіри.

---

### Модуль "Сповіщення"

**Призначення:** Автоматичні повідомлення клієнтам про зміни статусу рейсу та важливі події.

**Ключові дії:**
- автоматична відправка сповіщень при зміні статусу
- сповіщення про зміну ETA
- сповіщення про доставку
- налаштування каналів сповіщень (SMS, email, месенджери)

**Цінність для користувача:** Зменшення навантаження на експедитора, клієнт отримує інформацію без дзвінків, підвищення рівня сервісу.

---

### Модуль "Документи"

**Призначення:** Централізоване зберігання всіх документів по рейсу.

**Ключові дії:**
- завантаження документів до рейсу (ТТН, договори, фото, митні документи)
- перегляд та завантаження документів
- організація документів за типами
- доступ до документів через клієнтський лінк

**Цінність для користувача:** Всі документи в одному місці, швидкий доступ, зменшення ризику втрати документів, можливість швидко надати документи клієнту.

---

### Модуль "AI-відхилення"

**Призначення:** Аналіз відхилень від плану та рекомендації щодо оптимізації.

**Ключові дії:**
- виявлення аномалій у рейсах (затримки, простої, відхилення від маршруту)
- аналіз історичних даних для прогнозування проблем
- рекомендації щодо уникнення простоїв
- звіти про відхилення

**Цінність для користувача:** Проактивне виявлення проблем, рекомендації для оптимізації, зменшення простоїв та затримок.

---

### Модуль "Клієнтський лінк"

**Призначення:** Публічний доступ для клієнта до інформації про його рейси без реєстрації в системі.

**Ключові дії:**
- генерація унікального посилання для клієнта
- перегляд статусу рейсів клієнтом
- перегляд ETA та документів
- отримання сповіщень через лінк

**Цінність для користувача:** Клієнт має прозорість без дзвінків експедитору, експедитор зменшує навантаження на комунікації, підвищення рівня сервісу.

---

### Модуль "Аналітика / Дашборд керівника"

**Призначення:** Менеджерський дашборд з аналітикою для керівників компаній та відділів логістики.

**Ключові дії:**
- перегляд загальної статистики (кількість рейсів, середній час доставки, відсоток затримок)
- аналіз відхилень та проблемних маршрутів
- звіти про надійність перевізників
- рекомендації AI-модуля для оптимізації
- контроль SLA та KPI команди

**Цінність для користувача:** Об'єктивна картина виконання рейсів, можливість приймати рішення на основі даних, контроль якості роботи команди, виявлення проблемних зон.

**Основні метрики:**
- Кількість рейсів за період
- Середній час доставки
- Відсоток затримок
- Точність ETA
- Adoption клієнтського лінку

> **Примітка:** Цей модуль згадується в сценаріях використання (див. [/product/use-cases.md](/product/use-cases.md#сценарій-4-керівник-переглядає-статистику)). Планується до розробки на етапі V2 (див. [/strategy/roadmap.md](/strategy/roadmap.md#v2-масштабоване-рішення)).

## Взаємодія між модулями

Модулі DTLogix працюють як інтегрована система:

- **Рейси** — центральний модуль, до якого прив'язані всі інші модулі (ETA, Сповіщення, Документи, Клієнтський лінк)
- **ETA** автоматично оновлюється при зміні статусу рейсу та інтегрується з **Сповіщеннями**
- **Сповіщення** автоматично відправляються при змінах в модулях **Рейси** та **ETA**
- **Документи** прив'язані до конкретних рейсів та доступні через **Клієнтський лінк**
- **AI-модуль** аналізує дані з **Рейсів**, **ETA** та **Аналітики** для генерації рекомендацій
- **Аналітика** збирає дані з усіх модулів для формування звітів та статистики

Детальніше про технічну архітектуру: [/product/architecture.md](/product/architecture.md).

## Гіпотези

- Модуль рейсів — базовий, без нього система не працює.
- Модуль сповіщень критично важливий для adoption клієнтами експедиторів.
- AI-модуль стає цінним після накопичення достатньої кількості даних (1000+ рейсів).
- Клієнтський лінк зменшує навантаження на експедитора на 50–60%.

- Тестування інтеграції між модулями на реальних сценаріях.
- Збір метрик використання кожного модуля для оптимізації.
