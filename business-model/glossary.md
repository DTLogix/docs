# Глосарій термінів DTLogix

Цей документ містить визначення ключових термінів, що використовуються в документації DTLogix. Для детальних метрик та unit economics див. [/business-model/unit-economics.md](/business-model/unit-economics.md).

---

## CAC (Customer Acquisition Cost)

**Що це означає:**  
Вартість залучення одного платного клієнта. CAC показує, скільки грошей витрачає продукт, щоб отримати одного клієнта, який реально заплатив за сервіс.

**Формула:**  
`CAC = (Усі витрати на залучення клієнтів) / (Кількість нових платних клієнтів)`

**Де і як використовується в DTLogix:**  
- Ключова метрика для оцінки ефективності маркетингу та продажів
- Використовується для розрахунку unit economics (LTV:CAC)
- Варіюється за сегментами (Сегмент A: $20-30, Сегмент B: $30-50)
- Детальніше: [/business-model/unit-economics.md](/business-model/unit-economics.md)

---

## LTV (Lifetime Value)

**Що це означає:**  
Загальна вартість клієнта за весь період його використання продукту.

**Формула:**  
`LTV = ARPU × Середній термін життя клієнта`

**Де і як використовується в DTLogix:**  
- Ключова метрика для оцінки довгострокової цінності клієнта
- Використовується разом з CAC для оцінки unit economics (LTV:CAC)
- Варіюється за сегментами (Сегмент A: $480-1080, Сегмент B: $1080-2400)
- Детальніше: [/business-model/unit-economics.md](/business-model/unit-economics.md)

---

## ARPU (Average Revenue Per User)

**Що це означає:**  
Середній дохід на користувача або компанію за період (зазвичай місяць).

**Формула:**  
`ARPU = Загальний дохід / Кількість активних користувачів (або компаній)`

**Де і як використовується в DTLogix:**  
- Використовується для розрахунку LTV
- Варіюється за сегментами (Сегмент A: $40-60/міс, Сегмент B: $60-100/міс)
- Ключова метрика для прогнозування MRR та річного доходу
- Детальніше: [/business-model/unit-economics.md](/business-model/unit-economics.md)

---

## Churn rate

**Що це означає:**  
Відсоток клієнтів, які припиняють використання продукту за певний період (зазвичай місяць).

**Формула:**  
`Churn rate = (Кількість втрачених клієнтів за період / Кількість клієнтів на початку періоду) × 100%`

**Де і як використовується в DTLogix:**  
- Ключова метрика для оцінки retention та якості продукту
- Цільові значення: < 5%/міс для MVP, < 3%/міс для V2
- Зв'язаний з LTV: вищий churn → нижчий LTV
- Детальніше: [/business-model/8-key-metrics.md](/business-model/8-key-metrics.md)

## RICE

**Що це означає:**  
Framework для пріоритизації функцій та завдань на основі чотирьох факторів: Reach, Impact, Confidence, Effort.

**Компоненти:**
- **Reach** — скільки користувачів зачепить функція за період
- **Impact** — вплив на метрику (0.25 = мінімальний, 0.5 = низький, 1 = середній, 2 = високий, 3 = максимальний)
- **Confidence** — впевненість в оцінках (50% = низька, 80% = середня, 100% = висока)
- **Effort** — зусилля в людино-тижнях

**Формула:**  
`RICE Score = (Reach × Impact × Confidence) / Effort`

**Де і як використовується в DTLogix:**  
- Використовується для пріоритизації функцій у процесах розробки
- Допомагає об'єктивно оцінити вартість та цінність кожної функції
- Детальніше: [/processes/development.md](/processes/development.md#rice-framework)

## Unit economics

**Що це означає:**  
Модель оцінки економічної ефективності продукту на рівні одного клієнта, включає співвідношення LTV:CAC, payback period та інші метрики.

**Ключові компоненти:**
- LTV (Lifetime Value)
- CAC (Customer Acquisition Cost)
- LTV:CAC (співвідношення)
- ARPU (Average Revenue Per User)
- Payback period

**Де і як використовується в DTLogix:**  
- Ключова метрика для оцінки масштабованості бізнесу
- Цільове співвідношення LTV:CAC ≥ 3:1 для V1, ≥ 5:1 для V2
- Використовується для прийняття рішень про інвестиції в маркетинг та розробку
- Детальніше: [/business-model/unit-economics.md](/business-model/unit-economics.md)

## Клієнтський лінк (Client Link)

**Що це означає:**  
Публічне посилання для клієнта експедитора, через яке він може переглядати статус своїх рейсів, ETA та отримувати сповіщення без реєстрації в системі DTLogix.

**Де і як використовується в DTLogix:**  
- Дозволяє клієнтам експедиторів (Сегмент C) отримувати прозорість без дзвінків
- Зменшує навантаження на експедиторів (ціль: зменшення дзвінків на 50-60%)
- Критично важливий для adoption продукту експедиторами
- Цільова adoption rate: ≥ 50% рейсів
- Детальніше: [/product/modules.md](/product/modules.md#модуль-клієнтський-лінк)
