# Рішення (Solution)

## Вступ

Рішення повинно напряму закривати болі, зазначені у блоці [Проблеми (Problems)](/business-model/1-problems), і мати чітку логіку розвитку.

## Детальний опис

### MVP включає:

- створення заявки
- трекінг рейсу
- автоматичні статуси (loading / transit / customs / delivered)
- ETA
- фото та документи в одному місці
- автоматичні сповіщення клієнту
- базовий AI-модуль для аналізу відхилень

### Подальший розвиток:

- мобільний застосунок для водія
- API / webhooks
- GPS інтеграції
- менеджерський дашборд

## Пріоритизація MVP

**Must-have для першої комерційної версії:**

- створення заявки
- трекінг рейсу
- автоматичні статуси
- ETA
- автоматичні сповіщення клієнту
- зберігання фото та документів по рейсу

**Nice-to-have (можуть доїхати другим етапом):**

- базовий AI-модуль для аналізу відхилень
- розширені сценарії документів
- частина інтеграцій

## Гіпотези

- MVP достатньо для першої комерційної версії.
- Користувачам потрібно максимум 2–3 хвилини на створення рейсу.
- AI-рекомендації збільшать retention.

## Наступні дії / наслідки

- Створити прототип статусів і ETA.
- Перевірити сценарії реальних рейсів з 2–3 експедиторами.
- Зібрати дані для навчання AI-модуля.
- Зафіксувати критерій переходу до етапу росту: не менше 10 активних компаній та 200+ рейсів/місяць на платформі.
