# Рішення (Solution)

## Вступ

Рішення повинно напряму закривати болі, зазначені у блоці [Проблеми (Problems)](./1-problems.md), і мати чітку логіку розвитку.

## Детальний опис

### MVP включає:

- створення заявки  
- трекінг рейсу  
- автоматичні статуси (loading / transit / customs / delivered)  
- ETA  
- фото та документи в одному місці  
- автоматичні сповіщення клієнту  
- базовий AI-модуль для аналізу відхилень  

### Подальший розвиток:

- мобільний застосунок для водія  
- API / webhooks  
- GPS інтеграції  
- менеджерський дашборд  

## Гіпотези

- MVP достатньо для першої комерційної версії.  
- Користувачам потрібно максимум 2–3 хвилини на створення рейсу.  
- AI-рекомендації збільшать retention.  

## Наступні дії / наслідки

- Створити прототип статусів і ETA.  
- Перевірити сценарії реальних рейсів з 2–3 експедиторами.  
- Зібрати дані для навчання AI-модуля.
